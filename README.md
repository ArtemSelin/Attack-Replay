# Attack Replay (v2.0.0)

Attack Replay – это универсальный инструмент командной строки для тестирования функций безопасности различных сетевых средств защиты информации (СЗИ), таких как **NGFW** (Firewall, IDS/IPS, потоковый антивирус, DPI), **NDR**, **SIEM** и других решений сетевой безопасности.

## 🚀 Принцип работы

Программа принимает на вход заранее подготовленный файл или директорию с файлами в формате `pcap` / `pcapng` с записанным вредоносным или легитимным трафиком. После этого инструмент воспроизводит этот трафик с выбранного сетевого интерфейса в режиме **stateless** (без установления TCP-соединения). Attack Replay не предъявляет никаких требований к записанному трафику поэтому он позволяет воспроизводить любой тип трафика (в том числе и трафик, в котором используются нестандартные протоколы передачи данных).

Attack Replay можно применять как отдельно, так и во время нагрузочных тестов с целью проверки эффективности выявления и блокирования атак средствами защиты под нагрузкой.

Инструмент объединяет в себе возможности и гибкость таких утилит, как: `tcpreplay`, `tcprewrite`, `tcpprep` и `tcpreplay-edit` `tcpdump`.

## ✨ Основные возможности

- **Редактирование pcap-файлов**  
  – Замена MAC-адресов источника и назначения  
  – Замена IP-адресов источника и назначения  
  – Сохранение оригиналов (`-k`)

- **Воспроизведение трафика**  
  – Поддержка одного файла или целой директории  
  – Контроль скорости: `--topspeed`, `--pps`, `--mbps`, `--multiplier`  
  – Режимы работы: однократно, счётчик повторов (`-c`), бесконечный цикл (`-l`), по времени (`-t`)  
  – Настройка интервала между файлами и между циклами (`-i`)  
  – Тихий режим (`-q`) и короткая статистика (`--stats`)  
  – Усечение пакетов с MTU > 1500 (`--truncate`)  
  – Поддержка `--pktlen` (для старых/битых pcap)

- **Статистика**  
  – По каждому файлу: количество пакетов, байт, реальное время отправки  
  – Итоговая статистика: суммарные пакеты/байты, средняя скорость (Bps, Mbps, pps)

- **Вспомогательные утилиты**  
  – Показать сетевые интерфейсы (`-s`)  
  – Определить MAC по IP (`-a`)  
  – Автоматический подбор интерфейса и MAC по IP назначения (`--ip`)

## 🛠 Требования

Инструмент работает в Linux-окружении (Debian, Ubuntu, Kali Linux и др.). Для работы требуются системные утилиты:
* `tcpreplay` / `tcprewrite` / `tcpprep`
* `file`
* `bc`

При первом запуске скрипт автоматически проверяет наличие утилит и предлагает установить их через пакетный менеджер системы (`apt`, `yum`, `dnf`, `pacman`, `zypper`).
> ⚠️ **Важно:** для автоматической установки компонентов требуются права суперпользователя (**`sudo`**).

### Быстрый старт
Склонируйте репозиторий, назначьте права на исполнение и запустите интерактивную справку одной командой:

```bash
git clone https://github.com/ArtemSelin/Attack-Replay.git
cd Attack-Replay
chmod +x attack_replay.sh
sudo ln -s $(pwd)/attack_replay.sh /usr/local/bin/attack_replay
sudo attack_replay --help 
``````
### 📖 Описание синтаксиса и параметров
Синтаксис программы Attack Replay представляет собой:
```bash
./attack_replay.sh [параметры] <путь_к_pcap_файлу_или_директории>
```
