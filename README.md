# Attack Replay (v2.0.0)

Attack Replay – это универсальный инструмент командной строки для тестирования функций безопасности различных сетевых средств защиты информации (СЗИ), таких как **NGFW** (Firewall, IDS/IPS, потоковый антивирус, DPI), **NDR**, **SIEM** и других решений сетевой безопасности.

## 🚀 Принцип работы

Программа принимает на вход заранее подготовленный файл или директорию с файлами в формате `pcap` / `pcapng` с записанным вредоносным или легитимным трафиком. После этого инструмент воспроизводит этот трафик с выбранного сетевого интерфейса в режиме **stateless** (без установления TCP-соединения). Attack Replay не предъявляет никаких требований к записанному трафику поэтому он позволяет воспроизводить любой тип трафика (в том числе и трафик, в котором используются нестандартные протоколы передачи данных).

Attack Replay можно применять как отдельно, так и во время нагрузочных тестов с целью проверки эффективности выявления и блокирования атак средствами защиты под нагрузкой.

Инструмент объединяет в себе возможности и гибкость таких утилит, как: `tcpreplay`, `tcprewrite`, `tcpprep` и `tcpreplay-edit` `tcpdump`.

## ✨ Основные возможности

- **Редактирование pcap-файлов**  
  – Замена MAC-адресов источника и назначения  
  – Замена IP-адресов источника и назначения  
  – Сохранение оригиналов (`-k`)

- **Воспроизведение трафика**  
  – Поддержка одного файла или целой директории  
  – Контроль скорости: `--topspeed`, `--pps`, `--mbps`, `--multiplier`  
  – Режимы работы: однократно, счётчик повторов (`-c`), бесконечный цикл (`-l`), по времени (`-t`)  
  – Настройка интервала между файлами и между циклами (`-i`)  
  – Тихий режим (`-q`) и короткая статистика (`--stats`)  
  – Усечение пакетов с MTU > 1500 (`--truncate`)  
  – Поддержка `--pktlen` (для старых/битых pcap)

- **Статистика**  
  – По каждому файлу: количество пакетов, байт, реальное время отправки  
  – Итоговая статистика: суммарные пакеты/байты, средняя скорость (Bps, Mbps, pps)

- **Вспомогательные утилиты**  
  – Показать сетевые интерфейсы (`-s`)  
  – Определить MAC по IP (`-a`)  
  – Автоматический подбор интерфейса и MAC по IP назначения (`--ip`)

## 🛠 Требования

Инструмент работает в Linux-окружении (Debian, Ubuntu, Kali Linux и др.). Для работы требуются системные утилиты:
* `tcpreplay` / `tcprewrite` / `tcpprep`
* `file`
* `bc`

При первом запуске скрипт автоматически проверяет наличие утилит и предлагает установить их через пакетный менеджер системы (`apt`, `yum`, `dnf`, `pacman`, `zypper`).
> ⚠️ **Важно:** для автоматической установки компонентов требуются права суперпользователя (**`sudo`**).

### Быстрый старт
Склонируйте репозиторий, назначьте права на исполнение и запустите интерактивную справку одной командой:

```bash
git clone https://github.com/ArtemSelin/Attack-Replay.git
cd Attack-Replay
chmod +x attack_replay.sh
sudo ln -s $(pwd)/attack_replay.sh /usr/local/bin/attack_replay
sudo attack_replay --help 
```

## 📖 Описание синтаксиса и параметров
Синтаксис программы Attack Replay представляет собой:
```bash
attack_replay [параметры] <путь_к_pcap_файлу_или_директории>
```
### Основные опции
| Группа | Параметр | Описание |
|--------|----------|----------|
| **Редактирование** | `--src-mac=<MAC>` | Изменить MAC-адрес источника |
| | `--dst-mac=<MAC>` | Изменить MAC-адрес назначения |
| | `--src-ip=<IP>` | Изменить IP-адрес источника |
| | `--dst-ip=<IP>` | Изменить IP-адрес назначения |
| | `-k, --keep-original` | Сохранять оригиналы pcap |
| **Воспроизведение** | `--intf <интерфейс>` | Сетевой интерфейс отправки трафика |
| | `--ip <IP>` | Автоопределение интерфейса и MAC по IP-адресу цели |
| | `-i, --interval <сек>` | Пауза между отправкой файлов (0 — без паузы) |
| | `-l, --loop` | Бесконечный цикл воспроизведения трафика |
| | `-t, --time <сек>` | Воспроизведение в течение N секунд |
| | `-c, --count <число>` | Количество повторений набора файлов |
| **Скорость** | `--topspeed` | Максимально возможная скорость отправки |
| | `--pps <число>` | Ограничение количества пакетов в секунду |
| | `--mbps <число>` | Ограничение скорости в Мбит/с |
| | `--multiplier <число>` | Множитель оригинальной скорости |
| | `--oneatatime` | По одному пакету (Enter) |
| **Дополнительно** | `--stats` | Вывод краткой статистики (Actual / Rated) |
| | `--limit <число>` | Ограничить число отправляемых пакетов |
| | `--pktlen` | Использовать реальную длину пакета (не рекомендуется!) |
| | `--truncate` | Обрезать пакеты до стандартного MTU (1500 байт) |
| | `-v, --verbose` | Детальный вывод пакетов (вывод tcpdump) |
| | `-q, --quiet` | Тихий режим вывода статистики |
| **Вспомогательные команды** | `-s, --show-interfaces` | Вывести список доступных интерфейсов, их IP и MAC |
| | `-a, --arp-resolve` | Определить MAC по IP |
| | `-V, --version` | Версия программы |
| | `-h, --help` | Справка |

##💡 Примеры сценариев использования
### 1. Подготовка файлов (смена MAC-адресов)
Перед воспроизведением pcap (pcapng) файлов обязательно необходимо сменить MAC-адреса источника (сервер воспроизведения трафика с помощью attack_replay) и назначения (тестируемое средство защиты, например NGFW), в ином случае принимающая сторона отбросит входящий трафик, так как принимающее устройство проверяет MAC-адрес назначения, который не будет совпадать с MAC-адресом принимающего сетевого интерфейса (это стандартное поведение сетевых устройств – если пакеты не адресованы устройству, то они будут отброшены).
Изменение MAC-адресов во всех pcap-файлах каталога pcaps/ без сохранения оригиналов:
```bash
./attack_replay.sh --src-mac=08:00:27:d1:55:07 --dst-mac=08:00:27:18:1f:59 pcaps/
```
> ⚠️ **Примечание:** изменение MAC-адресов в pcap-файлах необходимо не для всех тестируемых средств защиты (в том числе и сетевых устройств), а только для конечных устройств, для которых предназначен воспроизводимый трафик (устройства, которые не маршрутизируют трафик или и не работают в неразборчивом режиме).
### 2. Воспроизведение pcap-файлов по таймеру со скоростью 5000 pps
Воспроизведение всех pcap файлов из каталога test/ через интерфейс eth0 в течение 6 минут (360 секунд) с паузой 10 секунд между циклами (тест RFC 9411 для NGFW на эффективности выявления и блокирования признаков компьютерных атак при проведении нагрузочного тестирования):
```bash
attack_replay "test/" --intf eth0 --pps 5000 -t 360 -i 10
```
### 3. Автоматический режим, редактирование и воспроизведение pcap файла в одно действие по IP-адресу цели
Программа сама определит нужный сетевой интерфейс, узнает локальный MAC-адрес, отправит ARP-запрос для выяснения MAC-адреса устройства 192.168.1.1, незаметно подставит эти данные "на лету" в заголовки пакетов и запустит генерацию на максимальной скорости:
```bash
sudo ./attack_replay.sh capture.pcap --ip 192.168.1.1 --topspeed
```
### 4. Генерация трафика в циклическом режиме
Генерация трафика со скоростью 3 Мбит/с в бесконечном циклическом режиме без пауз между файлами с выводом короткой статистики по отправляемому трафику:
```bash
./attack_replay.sh "pcaps/" --intf enp0s9 --mbps 3 -l -i 0 --stats
```
